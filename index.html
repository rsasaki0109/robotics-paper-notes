#1 Localization GNSS Factor Graph

GNSS/Multi-Sensor Fusion Using Continuous-Time Factor Graph Optimization for Robust Localization

GNSS + LiDAR + factor graph で、都市部ローカライゼーションに直結。最初に読むならこれ。

Why now

dense urban で classic fusion が破綻しやすい問題に、オンライン連続時間 FGO で正面から答えている。

What to learn

GNSS raw observation の扱い方、tight coupling の設計、実運用寄り localization の発想。

Detail note arXiv
#2 LiDAR-IMU EqF State Estimation

Equivariant Filter for Tightly Coupled LiDAR-Inertial Odometry

LiDAR-IMU の tightly coupled 推定を、EqF でどう安定化するかを短距離で学べる一本。

Why now

EKF 系で苦しみやすい非線形性と線形化誤差に対して、対称性ベースの設計がどう効くかが見える。

What to learn

IMU bias, navigation state, LiDAR extrinsic をまとめて扱う推定器設計と、幾何寄りの安定化の考え方。

Detail note arXiv
#3 LiDAR Odometry ICP Degeneracy

GenZ-ICP: Generalizable and Degeneracy-Robust LiDAR Odometry Using an Adaptive Weighting

corridor 退化や ill-posed optimization に強い ICP。現場で困る問題にちゃんと刺さる。

Why now

LiDAR odometry の破綻ケースは実運用で避けにくい。そこを adaptive weighting で抑える方向は実装価値が高い。

What to learn

degeneracy を意識した weighting 設計、ロバストな ICP への現実的な拡張、評価の見方。

Detail note arXiv
#4 VINS Dynamic Scene Robust Estimation

DynaVINS++: Robust Visual-Inertial State Estimator in Dynamic Environments by Adaptive Truncated Least Squares and Stable State Recovery

動的物体に壊される VINS を、学習ではなくロバスト推定と bias recovery で立て直す。

Why now

dynamic environment への対策は AI に寄りがちだが、これは classical な推定器改善として読めるのが良い。

What to learn

adaptive truncated least squares、bias consistency check、突然動く物体に対する VINS 保護策。

Detail note arXiv
#5 Calibration VIO Online

Universal Online Temporal Calibration for Optimization-Based Visual-Inertial Navigation Systems

時間ずれ補正は地味だけど効く。online temporal calibration を最初に押さえるならここ。

Why now

実機では sensor time offset が安定性を壊しやすい。しかも気づきにくい。だから先に読む価値が高い。

What to learn

optimization-based VINS に temporal calibration をどう入れるか、収束とノイズ耐性の見方。

Detail note arXiv
#6 IMU Preintegration Galilean Group

Equivariant IMU Preintegration with Biases: A Galilean Group Approach

IMU preintegration は localization 系の基礎ブロック。bias を含めた幾何的整理がきれい。

Why now

派手さはないが、VIO / INS / factor graph 系を読むときの共通言語として効く。

What to learn

bias を独立に扱わない preintegration、consistency 向上、Lie group ベースの見方。

Detail note arXiv
#7 Optimization Gaussian Mixture NLS

A Hessian for Gaussian Mixture Likelihoods in Nonlinear Least Squares

SLAM の表面ではなく、推定と最適化の地力を上げるタイプの一本。

Why now

Gaussian mixture likelihood を NLS に落とす時の Hessian 近似は、精度よりもまず理解コストに対するリターンが大きい。

What to learn

収束性、uncertainty characterization、推定器の内部をどう改善するかという視点。

Detail note arXiv
#8 LiDAR Place Recognition Narrow FOV

Narrowing Your FOV with SOLiD: Spatially Organized and Lightweight Global Descriptor for FOV-Constrained LiDAR Place Recognition

狭視野 LiDAR の place recognition。loop closure や再局在化の勘所を軽量に学びやすい。

Why now

実機では FOV 制約が普通にある。だから“理想センサ前提ではない” place recognition は見ておく価値がある。

What to learn

rotational change への対処、vertical information を使った reweighting、軽量 global descriptor の作り方。

Detail note arXiv
#9 Dense SLAM GPU Mapping

CoVoxSLAM: GPU Accelerated Globally Consistent Dense SLAM

dense SLAM を GPU で現実的に回す話。精度だけでなく実行性能に意識があるのが良い。

Why now

“動くかどうか” まで含めて dense SLAM を考えたいときに、GPU 活用の設計観点をつかみやすい。

What to learn

globally consistent volumetric SLAM、platform 差、discrete GPU と embedded GPU の見方。

Detail note arXiv
#10 Path Planning Geometry Global Optimality

Exact Wavefront Propagation for Globally Optimal One-To-All Path Planning on 2D Cartesian Grids

幾何ベースで global optimality を出す planning。基礎体力を上げるのに向いている。

Why now

近似で済ませがちな path planning を、exact wavefront propagation でどう解くかが明快で、整理しやすい。

What to learn

Eikonal 系との差、visibility ベースの解析的距離関数、one-to-all 計画の旨味。

Detail note arXiv
#11 MPC Planning Control

Variable Time-Step MPC for Agile Multi-Rotor UAV Interception of Dynamic Targets

可変 time-step と horizon の設計が面白い。MPC をもう一歩実用に寄せる発想が取れる。

Why now

MPC は使えるけれど重い。その制約を variable time-step でどうほぐすかは planning/control 共通の学びになる。

What to learn

prediction horizon と sampling density のトレードオフ、長区間と急機動を両立させる設計。

Detail note arXiv
#12 Distributed SLAM Certifiable Range-Aided

Distributed Certifiably Correct Range-Aided SLAM

すぐ現場投入するタイプではないが、distributed / certifiable SLAM の頭を整理するには強い。

Why now

range sensor + multi-agent + communication constraint という条件で、global optimum をどう扱うかの見取り図になる。

What to learn

Riemannian Staircase、distributed pose graph optimization から RA-SLAM への拡張、certifiable design の意味。

Detail note arXiv
